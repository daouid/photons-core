The Executor¶

This is where the mainline sits and is responsible for setting up the logging, the argument parsing and for starting up the App.

class photons_app.executor.App¶

The app is based on delfick-app and is responsible for several things:

  • Reading in environment variables
  • Reading in positional and keyword commandline arguments
  • Getting a target from the task argument
  • Making photons silent for the list_tasks and help tasks
  • Setting up logging
  • Creating and starting the Collector
  • Starting the chosen task
  • Cleanly stopping the asyncio event loop

Environment variables we look at are:

LIFX_CONFIG
Sets a value for –config and defaults to ‘./lifx.yml’

This class is extended in Photons Apps to provide custom startup functionalities.

Delfick App and Photons Executor provide the following commandline arguments:

usage: photons-docs [-h]
                    [--verbose | --silent | --debug | --logging-program LOGGING_PROGRAM]
                    [--tcp-logging-address TCP_LOGGING_ADDRESS]
                    [--udp-logging-address UDP_LOGGING_ADDRESS]
                    [--syslog-address SYSLOG_ADDRESS] [--json-console-logs]
                    [--version] [--dry-run] [--task PHOTONS_APP_CHOSEN_TASK]
                    [--artifact PHOTONS_APP_ARTIFACT]
                    [--reference PHOTONS_APP_REFERENCE]
                    [--config PHOTONS_APP_CONFIG]

Photons server!

optional arguments:
  -h, --help            show this help message and exit
  --verbose             Enable debug logging
  --silent              Only log errors
  --debug               Debug logs
  --logging-program LOGGING_PROGRAM
                        The program name to use when not logging to the
                        console
  --tcp-logging-address TCP_LOGGING_ADDRESS
                        The address to use for giving log messages to tcp
                        (i.e. localhost:9001)
  --udp-logging-address UDP_LOGGING_ADDRESS
                        The address to use for giving log messages to udp
                        (i.e. localhost:9001)
  --syslog-address SYSLOG_ADDRESS
                        The address to use for syslog (i.e. /dev/log)
  --json-console-logs   If we haven't set other logging arguments, this will
                        mean we log json lines to the console
  --version             Print out the version!
  --dry-run             Should we take any real action or print out what is
                        intends to do
  --task PHOTONS_APP_CHOSEN_TASK
                        The task to run
  --artifact PHOTONS_APP_ARTIFACT
                        Extra information
  --reference PHOTONS_APP_REFERENCE
                        A selector for the lights
  --config PHOTONS_APP_CONFIG
                        Config file to read from
