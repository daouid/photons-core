photons_colour¶

This module knows about colour.

TASK:: set_color

Change specified bulb to specified colour

lan:set_color d073d50000 red -- '{"hue": 205}'

The format of this task is <reference> <color> -- <overrides> where overrides is optional.

The color may be any valid color specifier.

These make use of the following helpers:

          )
    , ('kelvin', T.Uint16.default(3500))
    )

duration_typ = T.Uint32.default(0).transform(

color_and_duration = (

waveform_opts = (

              lambda _, value: int(1000 * float(value))
            , lambda value: float(value) / 1000
            ).allow_float()
          )
    , ('cycles', T.Float.default(1))
    , ('skew_ratio', T.Int16.default(0).transform(

    , ('waveform', T.Uint8.enum(Waveform).default(Waveform.SAW))
    )

class photons_colour.Parser¶

Class for converting valid colour specifiers into the options for a SetWaveFormOptional device command.

Note

http://colorizer.org/ and http://colormine.org/ are amazing!

The valid colour names are:

'blue' 'cyan' 'green' 'orange' 'pink' 'purple' 'red' 'white' 'yellow'

And valid colour specifiers are as follows:

01_hsb_component
\Ahsb:([\d\.]+),([\d\.]+%?),([\d\.]+%?)\Z
02_rgb_component
\Argb:(\d+,\d+,\d+)\Z
03_hex_component
\A(?:hex:)?#?([0-9a-fA-F]{6})\Z
04_hue_component
\Ahue:([\d.]+)\Z
05_saturation_component
\Asaturation:([\d.]+%?)\Z
06_brightness_component
\Abrightness:([\d.]+%?)\Z
07_kelvin_component
\Akelvin:([\d]+)\Z
10_random_component
random
whitespace
\s+

This class also has some classmethods for your use:
